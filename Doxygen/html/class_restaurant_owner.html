Detailed Description

Contains all methods for actions that occur by the owner of a restaurant for smooth operation.

References Customer with a high degree of integration. The suggested implementation is to facilitate most of the interaction with a Customer instance. Contains additional (standalone) methods for general restaurant upkeep.

◆ dirty_tables

int RestaurantOwner::dirty_tables

The total number of tables that the owner is tracking as dirty.

Suggested implentation involves incrementing this value after selling a food item to a Customer. Additonally, suggested implementation would be that this value never falls below 0, but this method will do nothing in the event this happens.

◆ total_sales

int RestaurantOwner::total_sales

The total profit that an owner has recieved from customers.

There is no suggested implementation for resetting this value, and it can be used to track daily, weekly, monthy, etc total sales depending on need.
